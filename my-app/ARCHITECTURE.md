## Поток аутентификации

```
┌─────────────────────────────────────────────────────────────┐
│                    РЕГИСТРАЦИЯ                              │
└─────────────────────────────────────────────────────────────┘

Пользователь → /register → RegisterPage
                ↓
         Вводит данные
                ↓
         AuthContext.register()
                ↓
         Firebase Auth
         createUserWithEmailAndPassword()
                ↓
         Firestore → users/{uid}
         Сохранение профиля
                ↓
         AuthContext.setUser()
                ↓
         Редирект → /profile
                ↓
         ✅ Успешная регистрация


┌─────────────────────────────────────────────────────────────┐
│                        ВХОД                                 │
└─────────────────────────────────────────────────────────────┘

Пользователь → /login → LoginPage
                ↓
         Вводит email и пароль
                ↓
         AuthContext.login()
                ↓
         Firebase Auth
         signInWithEmailAndPassword()
                ↓
         Firestore ← Загрузить профиль
         users/{uid}
                ↓
         AuthContext.setUser()
                ↓
         Редирект → /profile
                ↓
         ✅ Успешный вход


┌─────────────────────────────────────────────────────────────┐
│                 ДОСТУП К ЗАЩИЩЁННЫМ МАРШРУТАМ              │
└─────────────────────────────────────────────────────────────┘

Пользователь → Клик на /admin
                ↓
         ProtectedRoute проверка
                ↓
         ├─ isLoading? → Показать загрузку
                ↓
         ├─ user == null? → Редирект на /login
                ↓
         ├─ requiredRole='admin' && !user.isAdmin? 
                ↓
                → Редирект на /
                ↓
         └─ ✅ Все проверки пройдены
                ↓
         Отобразить компонент (Panel.tsx)
```

## Взаимодействие компонентов

```
App.tsx
├── AuthProvider (AuthContext.tsx)
│   ├── Управление состоянием пользователя
│   ├── Слушание изменений Firebase Auth
│   └── Загрузка данных профиля из Firestore
│
├── CartProvider (CartContext.tsx)
│   └── Управление корзиной пользователя
│
└── Routes
    ├── <Route path="/" element={<Home />} />
    │
    ├── <Route path="/login" element={<Login />} />
    │   └── useAuth() → login()
    │
    ├── <Route path="/profile" element={
    │   <ProtectedRoute>
    │       <Profile />
    │       └── useAuth() → user, updateUserProfile()
    │   </ProtectedRoute>
    │ }>
    │
    └── <Route path="/admin" element={
        <ProtectedRoute requiredRole="admin">
            <Panel /> (Admin.tsx)
            ├── Товары (управление товарами)
            ├── Заказы (просмотр заказов)
            └── Пользователи (UserManagement.tsx)
                ├── Загрузка всех пользователей
                ├── Редактирование профилей
                ├── Управление правами
                └── Удаление пользователей
        </ProtectedRoute>
    }>
```

## Firestore Security Rules - Иерархия доступа

```
┌─────────────────────────────────────────────────────────────┐
│          FIRESTORE SECURITY RULES HIERARCHY                 │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────┐
│   НЕАВТОРИЗОВАННЫЙ      │
│   ПОЛЬЗОВАТЕЛЬ          │
└─────────────────────────┘
         ↓
    ❌ Запрещено:
    - Читать профили пользователей
    - Писать/Редактировать данные
    - Удалять документы
         ↓
    ✅ Разрешено:
    - Читать товары (публичные)


┌─────────────────────────────────────────┐
│   АВТОРИЗОВАННЫЙ ПОЛЬЗОВАТЕЛЬ           │
│   (isAdmin = false)                     │
└─────────────────────────────────────────┘
         ↓
    ✅ Разрешено:
    - Читать свой профиль
    - Редактировать свой профиль
    - Читать/Писать свою корзину
    - Создавать свои заказы
    - Читать свои заказы
    - Читать все товары
         ↓
    ❌ Запрещено:
    - Читать профили других пользователей
    - Читать профили администраторов
    - Редактировать товары
    - Редактировать заказы
    - Удалять данные


┌────────────────────────────┐
│   АДМИНИСТРАТОР             │
│   (isAdmin = true)          │
└────────────────────────────┘
         ↓
    ✅ Разрешено:
    - Читать ВСЕ профили
    - Редактировать ВСЕ профили
    - Создавать/Редактировать товары
    - Удалять товары
    - Читать ВСЕ заказы
    - Обновлять статус заказов
    - Создавать новых администраторов
    - Читать логи и аналитику
         ↓
    ❌ Запрещено:
    - Удалять себя
    - Отозвать свои права администратора
```

                 Обновить таблицу


    ┌─── Управление правами ───┐

## Жизненный цикл сеанса пользователя

```
Пользователь открывает приложение
         ↓
    App.tsx инициализируется
         ↓
    AuthProvider монтируется
         ↓
    onAuthStateChanged() слушатель установлен
         ↓
    Firebase проверяет токен сеанса
         ↓
    ├─ Токен действителен?
    │   ├─ ✅ ДА
    │   │   ↓
    │   │   Загрузить профиль из Firestore
    │   │   ↓
    │   │   setState(user, isLoading=false)
    │   │   ↓
    │   │   ✅ Пользователь авторизован
    │   │
    │   └─ ❌ НЕТ
    │       ↓
    │       setState(user=null, isLoading=false)
    │       ↓
    │       ❌ Пользователь не авторизован
    │
    └─ Пользователь может использовать приложение
         ↓
    Клик на защищённый маршрут
         ↓
    ProtectedRoute проверяет:
    - isLoading?
    - user == null?
    - user.isAdmin?
         ↓
    ├─ Все проверки пройдены → Отобразить компонент
    └─ Проверка не пройдена → Редирект или загрузка


    Пользователь нажимает "Выход"
         ↓
    logout()
         ↓
    signOut(auth)
         ↓
    Firebase удаляет токен
         ↓
    setState(user=null)
         ↓
    Редирект на /login
         ↓
    ❌ Сеанс завершён
```

## Интеграция с React Router

```
Browser History
         ↓
    React Router
         ↓
    App.tsx Routes
         ↓
    ├─ Публичный маршрут?
    │   └─ Рендеринг компонента
    │
    ├─ ProtectedRoute?
    │   ├─ Проверка аутентификации
    │   ├─ Проверка ролей
    │   ├─ ✅ Все OK? → Рендеринг компонента
    │   └─ ❌ Ошибка? → Редирект
    │
    └─ Navigation изменяется
         ↓
    Компонент получает props (user, logout и т.д.)
         ↓
    useAuth() hook внутри компонента
         ↓
    Отрисовка UI в зависимости от состояния
```
